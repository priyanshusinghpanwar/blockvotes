HOW TO RUN BLOCKVOTES
=====================

Project root:
E:\Project\Block-Chain\Block-Chain

This project uses pnpm workspaces.


1. INSTALL DEPENDENCIES
-----------------------

Run from the project root:

  corepack enable
  pnpm install


2. SET UP THE BACKEND ENVIRONMENT
---------------------------------

The backend env file should be here:

  artifacts\api-server\.env

If it does not exist, copy the example:

  Copy-Item artifacts\api-server\.env.example artifacts\api-server\.env

Important values for local/demo use:

  DATABASE_URL=postgresql://postgres:postgres@localhost:5432/blockvotes
  PORT=3000
  APP_URL=https://blockvotes.vercel.app
  PUBLIC_WEB_URL=https://blockvotes.vercel.app
  CORS_ORIGIN=https://blockvotes.vercel.app,http://localhost,capacitor://localhost
  SESSION_COOKIE_SAME_SITE=none
  SESSION_COOKIE_SECURE=true


3. PREPARE THE DATABASE
-----------------------

Make sure PostgreSQL is running and the database exists:

  blockvotes

Then push the schema:

  pnpm --filter @workspace/db run push


4. RUN THE BACKEND API
----------------------

Open a terminal at the project root and run:

  pnpm --filter @workspace/api-server run dev

The API runs on:

  http://localhost:3000


5. RUN THE WEB FRONTEND
-----------------------

Open a second terminal at the project root and run:

  pnpm --filter @workspace/blockvotes run dev

Vite will print the frontend URL, usually:

  http://localhost:5173


6. RUN WITH NGROK FOR MOBILE OR DEPLOYED FRONTEND
-------------------------------------------------

If the Android app or deployed frontend needs to reach your local backend, keep the API running and open another terminal:

  ngrok http 3000

Use the public ngrok URL as the frontend API URL.

Current configured demo backend URL:

  https://screen-uniformly-scoured.ngrok-free.dev

If ngrok gives a new URL, rebuild the mobile app with:

  $env:VITE_API_BASE_URL = "https://YOUR-NEW-NGROK-URL"
  pnpm --filter @workspace/blockvotes run build:mobile


7. BUILD OR UPDATE THE ANDROID APP
----------------------------------

Install Android Studio first.

To regenerate app icons after changing resources\icon.png:

  pnpm --filter @workspace/blockvotes run cap:assets

To build and sync the web app into Android:

  pnpm --filter @workspace/blockvotes run build:mobile

To open the Android project:

  pnpm --filter @workspace/blockvotes run mobile:android

In Android Studio:

  1. Connect an Android phone with USB debugging enabled, or start an emulator.
  2. Select the device.
  3. Press Run.


8. CREATE A SHAREABLE APK
-------------------------

From the Android project folder:

  cd artifacts\blockvotes\android

If Java is not in PATH, use Android Studio's bundled Java:

  $env:JAVA_HOME = "C:\Program Files\Android\Android Studio\jbr"
  $env:Path = "$env:JAVA_HOME\bin;$env:Path"

Build the debug APK:

  .\gradlew.bat assembleDebug

The generated APK is:

  artifacts\blockvotes\android\app\build\outputs\apk\debug\app-debug.apk

The easy-to-share copy used in this project is:

  artifacts\blockvotes\BlockVotes-debug.apk

To refresh that copy after building:

  Copy-Item artifacts\blockvotes\android\app\build\outputs\apk\debug\app-debug.apk artifacts\blockvotes\BlockVotes-debug.apk -Force


9. QUICK CHECK COMMANDS
-----------------------

Typecheck everything:

  pnpm run typecheck

Build the frontend/mobile web bundle:

  pnpm --filter @workspace/blockvotes run build

Build the backend:

  pnpm --filter @workspace/api-server run build

Run backend tests:

  pnpm --filter @workspace/api-server run test


10. COMMON ISSUES
-----------------

If the APK installs but still shows the old icon:

  Uninstall the old BlockVotes app from the phone, then install the new APK again.

If Android build says JAVA_HOME is missing:

  Set JAVA_HOME to Android Studio's bundled JBR:

    C:\Program Files\Android\Android Studio\jbr

If the mobile app cannot connect to the backend:

  1. Make sure the API is running on port 3000.
  2. Make sure ngrok is running and forwarding to http://localhost:3000.
  3. Make sure VITE_API_BASE_URL uses the current ngrok URL.
  4. Make sure CORS_ORIGIN in artifacts\api-server\.env includes:

     http://localhost,capacitor://localhost

If ngrok changes URL:

  Update VITE_API_BASE_URL and rebuild the mobile app.
